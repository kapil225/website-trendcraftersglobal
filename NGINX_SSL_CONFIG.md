# Nginx SSL/HTTPS Configuration Guide

**DOMAIN:** `trendcrafters.global` / `www.trendcrafters.global`

**IMPORTANT:** This configuration assumes:
- Cloudflare SSL/TLS mode is set to "Full (strict)"
- Let's Encrypt certificates are installed at `/etc/letsencrypt/live/trendcrafters.global/`
- Gunicorn is running on `127.0.0.1:8000`
- Static files are at `/path/to/project/staticfiles/`
- Media files are at `/path/to/project/media/`

---

## Step 1: Nginx Configuration File

**File Location:** `/etc/nginx/sites-available/trendcrafters`

```nginx
# ============================================================================
# TRENDCRAFTERS NGINX CONFIGURATION - PRODUCTION SSL/HTTPS
# ============================================================================
# This configuration handles:
# - HTTP → HTTPS redirect (no redirect loops)
# - SSL/TLS with Let's Encrypt certificates
# - Cloudflare proxy compatibility
# - Static & media file serving
# - Gunicorn WSGI app routing

# ============================================================================
# HTTP SERVER BLOCK - REDIRECT ONLY
# ============================================================================
# Purpose: Catch all HTTP requests and redirect to HTTPS
# This prevents ERR_TOO_MANY_REDIRECTS issues
server {
    listen 80;
    listen [::]:80;
    
    server_name trendcrafters.global www.trendcrafters.global;
    
    # Certbot ACME challenge directory (for certificate renewal)
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }
    
    # Redirect all other HTTP traffic to HTTPS
    location / {
        # Use 301 (permanent) for SEO
        return 301 https://$server_name$request_uri;
    }
}

# ============================================================================
# HTTPS SERVER BLOCK - MAIN PRODUCTION SERVER
# ============================================================================
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    
    server_name trendcrafters.global www.trendcrafters.global;
    
    # ========================================================================
    # SSL CERTIFICATE CONFIGURATION (Let's Encrypt + Certbot)
    # ========================================================================
    # Paths generated by: certbot certonly --standalone -d trendcrafters.global
    ssl_certificate /etc/letsencrypt/live/trendcrafters.global/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/trendcrafters.global/privkey.pem;
    
    # Strongly Encrypted SSL
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    # SSL session cache for performance
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    ssl_session_tickets off;
    
    # ========================================================================
    # SECURITY HEADERS (Prevents mixed content, XSS, etc.)
    # ========================================================================
    
    # HSTS: Force HTTPS for next 1 year (ensures no mixed content)
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    
    # Block browser sniffing
    add_header X-Content-Type-Options "nosniff" always;
    
    # Prevent clickjacking
    add_header X-Frame-Options "DENY" always;
    
    # Enable XSS protection
    add_header X-XSS-Protection "1; mode=block" always;
    
    # Referrer policy
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    
    # ========================================================================
    # CLOUDFLARE COMPATIBILITY SETTINGS
    # ========================================================================
    # When Cloudflare is in "Full (strict)" mode:
    # - Client connects to Cloudflare over HTTPS
    # - Cloudflare connects to Nginx over HTTPS
    # - Nginx receives X-Forwarded-Proto: https header
    
    # Trust Cloudflare's X-Forwarded-For header to get real client IP
    # This is safe because we're behind Cloudflare's proxy
    set_real_ip_from 173.245.48.0/20;
    set_real_ip_from 103.21.244.0/22;
    set_real_ip_from 103.22.200.0/22;
    set_real_ip_from 103.31.4.0/22;
    set_real_ip_from 141.101.64.0/18;
    set_real_ip_from 108.162.192.0/18;
    set_real_ip_from 190.93.240.0/20;
    set_real_ip_from 188.114.96.0/20;
    set_real_ip_from 197.234.240.0/22;
    set_real_ip_from 198.41.128.0/17;
    set_real_ip_from 162.158.0.0/15;
    set_real_ip_from 104.16.0.0/12;
    set_real_ip_from 172.64.0.0/13;
    set_real_ip_from 2400:cb00::/32;
    set_real_ip_from 2606:4700::/32;
    set_real_ip_from 2803:f800::/32;
    set_real_ip_from 2405:b500::/32;
    set_real_ip_from 2405:8100::/32;
    set_real_ip_from 2a06:98c0::/29;
    set_real_ip_from 2c0f:f248::/32;
    real_ip_header CF-Connecting-IP;
    
    # ========================================================================
    # STATIC FILES (CSS, JS, Images)
    # ========================================================================
    # Django's whitenoise serves static files, but Nginx can serve them faster
    location /static/ {
        alias /path/to/project/staticfiles/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        add_header Access-Control-Allow-Origin "*";
        
        # Gzip compression for text files
        gzip on;
        gzip_types text/plain text/css text/javascript application/json;
    }
    
    # ========================================================================
    # MEDIA FILES (Uploaded content)
    # ========================================================================
    location /media/ {
        alias /path/to/project/media/;
        expires 7d;
        add_header Cache-Control "public";
    }
    
    # ========================================================================
    # DJANGO APPLICATION (Gunicorn)
    # ========================================================================
    location / {
        # Pass all requests to Gunicorn
        proxy_pass http://127.0.0.1:8000;
        
        # Pass original request information to Django
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # Cloudflare specific
        proxy_set_header CF-Connecting-IP $http_cf_connecting_ip;
        
        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
        
        # Buffer settings for large responses
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 24 4k;
        
        # WebSocket support (if needed)
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
    
    # ========================================================================
    # DENY ACCESS TO SENSITIVE FILES
    # ========================================================================
    location ~ /\.git {
        deny all;
    }
    
    location ~ /\.env {
        deny all;
    }
    
    location ~ /\.well-known/acme-challenge/ {
        root /var/www/certbot;
    }
    
    # ========================================================================
    # LOGGING
    # ========================================================================
    access_log /var/log/nginx/trendcrafters_access.log;
    error_log /var/log/nginx/trendcrafters_error.log;
}
```

## Step 2: Enable the Configuration

```bash
# Create symbolic link to enable the site
sudo ln -s /etc/nginx/sites-available/trendcrafters /etc/nginx/sites-enabled/

# Test Nginx configuration for syntax errors
sudo nginx -t

# If test passes, reload Nginx
sudo systemctl reload nginx

# Verify Nginx is running
sudo systemctl status nginx
```

## Step 3: Important Paths to Update

Replace the following paths in the Nginx config with your actual paths:

```bash
# Find your project directory
cd /path/to/project
pwd

# Find static files location
ls -la staticfiles/

# Find media files location
ls -la media/

# Find Gunicorn socket/port
ps aux | grep gunicorn
```

**Update these lines in the Nginx config:**

```nginx
# Line for static files
alias /path/to/project/staticfiles/;

# Line for media files
alias /path/to/project/media/;

# Line for Gunicorn (if using socket instead of port)
proxy_pass http://unix:/path/to/project/gunicorn.sock;
```

## Troubleshooting Nginx Configuration

### Check if Nginx listens on port 443

```bash
sudo lsof -i :443
```

### Check if Nginx listens on port 80

```bash
sudo lsof -i :80
```

### View Nginx error log for SSL issues

```bash
sudo tail -f /var/log/nginx/trendcrafters_error.log
```

### View Nginx access log

```bash
sudo tail -f /var/log/nginx/trendcrafters_access.log
```

### Verify SSL certificate is valid

```bash
sudo openssl x509 -in /etc/letsencrypt/live/trendcrafters.global/fullchain.pem -text -noout
```

## Port Configuration Summary

| Port | Protocol | Purpose | Handled By |
|------|----------|---------|-----------|
| 80   | HTTP     | Redirect to HTTPS only | Nginx |
| 443  | HTTPS    | Main application server | Nginx + Gunicorn |
| 8000 | HTTP     | Internal Gunicorn (not exposed) | Gunicorn |

**IMPORTANT:** Ensure AWS Security Groups allow:
- Inbound: Port 80 (HTTP)
- Inbound: Port 443 (HTTPS)
- Outbound: All (for updates, etc.)
